# Inventory Manager — Flutter + Firestore

A real-time inventory management app built with Flutter and Cloud Firestore for **CSC Mobile App Development, In-Class Activity #12**.

## Overview

This app lets users **create, read, update, and delete** inventory items, with the UI staying in sync with Firestore in real time via `StreamBuilder`. The codebase is split into a **model**, a **service layer**, and **UI screens** so that data logic stays out of the widgets.

## Features

### Core
- Firebase + Firestore integration, initialized in `main.dart`
- Add items with name, quantity, and price
- Real-time list powered by `StreamBuilder` and `ListView.builder`
- Edit items by tapping the edit icon
- Delete items with a confirmation dialog and snackbar feedback
- Form validation — required fields, numeric checks, no negatives
- UX states — loading spinner, empty state, error display

### Enhanced Features (the 2 required extras)

**1. Live search bar**
A search field at the top of the inventory screen filters items by name in real time as the user types. The filter runs client-side over the live Firestore stream, so results update instantly without any extra database reads. A clear (✕) button resets the search with one tap, and an empty-results message appears if no items match.

**2. Low-stock badges + multi-mode sort**
Any item with a quantity below 5 displays a red **LOW** badge directly in the list, making it easy to spot items that need restocking. A sort menu in the AppBar lets the user reorder the inventory by **Newest**, **Name (A–Z)**, **Quantity (ascending)**, or **Price (descending)**. A summary header above the list also shows the total item count and total inventory value in dollars, calculated from `quantity × price` across all items.

## Architecture
lib/
├── main.dart                    # Firebase init + MaterialApp
├── firebase_options.dart        # Auto-generated by flutterfire configure
├── models/
│   └── item.dart                # Item model + toMap/fromMap
├── services/
│   └── firestore_service.dart   # All CRUD logic (add/stream/update/delete)
└── screens/
├── inventory_screen.dart    # StreamBuilder list, search, sort
└── item_form_screen.dart    # Add/Edit form with validation

The **service layer** isolates all Firestore calls so that the UI never touches `FirebaseFirestore` directly. This makes the code easier to test, easier to swap backends later, and matches the *engineer with code quality* objective from the rubric.

## Setup

1. Clone the repo and install dependencies:
```bash
   git clone <your-repo-url>
   cd inventory_app
   flutter pub get
```
2. Configure Firebase for your own project:
```bash
   flutterfire configure
```
   This generates `lib/firebase_options.dart`.
3. Enable **Cloud Firestore** in your Firebase console (test mode is fine for development).
4. Run the app:
```bash
   flutter run
```

## Building the APK
```bash
flutter build apk --release
```
The APK will be at `build/app/outputs/flutter-apk/app-release.apk`.
